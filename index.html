Curso GIT de Pildoras Informáticas

Video 2

    • git init: para inicializar
      git status -s: para ver en terminal el estatus de los archivos en carpeta
      git add archivo: para indicar que se pasa el archivo al staging area
      git commit -m "comentario": para pasar el archivo al repositorio local incluyendo el comentario para indicar los cambios
      git log --oneline: verificar cuantas copias hay en el repositorio local
      git reset log --hard clave de la copia: cargar la copia realizada
      git add .: hacer seguimiento a toda la carpeta en que se esta trabajando
      git commit -am "comentario": hacer comit pasando el staging area
      git commit --amend: reemplazar el comentario del repositorio actual

    • git push: para enviar los cambios a Github

    git pull: para traer los cambios desde Github

    git tag descripción -m "Comentario"/p>

    git push --tags: envia las etiquetas (versiones) a Github

    git clone url: clona el repositorio de Github

    Ramas

    git branch nombre de la rama: crear una rama

    git branch: muestras las ramas existentes en el proyecto en verde indica en cual se esta trabajando

    git checkout nombre de la rama: cambiar de rama en donde se va a trabajar

    git marge nombre de la rama: fusiona los cambios de una rama en la rama principal. Primero moverse a la rama principal,
